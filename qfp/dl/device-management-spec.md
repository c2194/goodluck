# 设备管理功能描述

## 1. 功能背景

“祝福牌”是一种可远程更新内容的电子挂牌设备。每台祝福牌内部划分为若干个排位，每个排位可以分配给一个最终用户使用。最终用户可上传并更新自己排位对应的图片和音频内容，设备按既定顺序循环展示和播放各排位内容。

本功能旨在建立一套完整的“设备管理”能力，用于管理祝福牌、排位、最终用户与内容之间的关系，并支持内容远程更新、设备同步和状态跟踪。

## 2. 功能目标

1. 支持对祝福牌设备进行统一管理。
2. 支持为每台设备配置多个排位。
3. 支持将排位分配给具体最终用户。
4. 支持最终用户更新自己排位的图片和音频内容。
5. 支持设备定时或主动拉取最新内容配置。
6. 支持运营人员查看设备状态、排位分配情况和内容更新记录。

## 3. 核心对象定义

### 3.1 祝福牌设备

祝福牌设备是最终播放内容的硬件终端。每台设备具备唯一身份标识，并能周期性从服务器获取配置与内容。

设备主要属性建议包括：

1. 设备编号
2. 设备名称
3. 所属批次或月份
4. 通信标识（如 MAC、设备码）
5. 在线状态
6. 最后请求时间
7. 累计请求次数
8. 当前配置参数
9. 设备所属排位数量

### 3.2 排位

排位是祝福牌中的独立内容单元。每个排位对应一组图片、音频和展示状态，并可被单独分配给一个最终用户。

排位主要属性建议包括：

1. 排位编号
2. 所属设备
3. 当前状态
4. 分配的最终用户
5. 当前图片内容
6. 当前音频内容
7. 最近更新时间
8. 启用或禁用状态

### 3.3 最终用户

最终用户是排位内容的维护者。最终用户只能管理自己被分配的排位内容，不能操作其他用户或设备的内容。

最终用户主要属性建议包括：

1. 用户编号
2. 用户昵称或姓名
3. 联系方式
4. 所属排位
5. 可用状态
6. 最近上传时间

### 3.4 内容资源

内容资源是最终由设备播放的实际文件，主要包括图片和音频。

内容资源主要属性建议包括：

1. 资源类型
2. 文件路径
3. 文件大小
4. 上传时间
5. 所属排位
6. 版本号
7. 是否已发布

## 4. 用户角色

### 4.1 管理员

管理员负责设备、排位和用户的整体管理。管理员可进行以下操作：

1. 新增和管理祝福牌设备
2. 配置设备排位数量
3. 将排位分配给最终用户
4. 查看设备在线情况和请求记录
5. 审核或管理最终用户上传内容
6. 调整设备播放参数

### 4.2 最终用户

最终用户负责维护自己排位对应的展示内容。最终用户可进行以下操作：

1. 查看自己被分配的排位
2. 上传或替换图片
3. 上传或替换音频
4. 预览当前内容
5. 查看内容是否已生效

## 5. 核心业务流程

### 5.1 设备创建与排位初始化

1. 管理员新增一台祝福牌设备。
2. 系统为该设备初始化若干个排位。
3. 每个排位具备唯一编号，并处于“未分配”状态。

### 5.2 排位分配

1. 管理员选择某台祝福牌的某个排位。
2. 管理员将该排位绑定给某个最终用户。
3. 系统记录分配关系，并允许该用户开始上传内容。

### 5.3 最终用户上传内容

1. 最终用户登录后查看自己被分配的排位。
2. 用户上传图片和音频。
3. 系统完成文件校验、转码、缩略图或预览生成。
4. 系统将内容与对应排位绑定，并标记为最新版本。

### 5.4 设备获取内容

1. 祝福牌设备周期性请求服务端配置接口。
2. 服务端返回当前设备对应的排位内容清单及播放参数。
3. 设备依据返回数据下载或更新资源。
4. 设备按排位顺序循环播放内容。

### 5.5 内容更新生效

1. 当最终用户更新内容后，系统记录更新时间。
2. 设备下次请求配置时，自动获得最新内容。
3. 如内容已变更，设备下载新文件并替换本地旧文件。

## 6. 功能范围

### 6.1 设备管理

1. 新增设备
2. 编辑设备信息
3. 查看设备详情
4. 查看设备请求记录
5. 查看设备最后在线时间
6. 查看设备定位或基站信息

### 6.2 排位管理

1. 查看某设备下所有排位
2. 查看排位当前用户
3. 查看排位当前内容状态
4. 调整排位启用或禁用状态
5. 解绑或重新分配排位

### 6.3 最终用户内容管理

1. 上传图片
2. 上传音频
3. 替换已有内容
4. 删除内容
5. 预览内容
6. 查看最近更新时间

### 6.4 设备同步与播放控制

1. 下发图片与音频资源
2. 下发播放顺序与状态
3. 下发设备参数
4. 记录设备拉取配置的次数和时间

## 7. 关键业务规则

1. 一个排位同一时刻只能分配给一个最终用户。
2. 一个最终用户可被分配一个或多个排位，具体数量由业务规则决定。
3. 最终用户只能编辑自己所属排位的内容。
4. 设备只播放已启用且有有效内容的排位。
5. 新上传内容应覆盖旧版本，或保留历史版本供回溯。
6. 设备请求接口时，应记录访问次数和最后访问时间。
7. 内容更新后不要求设备立即在线，但下次设备拉取配置时必须能获取最新内容。

## 8. 管理端页面建议

### 8.1 设备列表页

建议展示以下信息：

1. 设备编号
2. 设备名称
3. 所属批次
4. 排位数量
5. 最后在线时间
6. 累计请求次数
7. 当前状态
8. 快速进入设备详情

### 8.2 设备详情页

建议展示以下信息：

1. 设备基本信息
2. 各排位分配情况
3. 各排位当前图片和音频
4. 排位状态
5. 最后请求时间
6. 累计请求次数
7. 基站信息或位置信息

### 8.3 用户内容管理页

建议展示以下信息：

1. 当前用户可管理的排位
2. 图片上传入口
3. 音频上传入口
4. 内容预览
5. 最近更新记录

## 9. 数据记录建议

### 9.1 设备维度

建议记录：

1. 设备基础信息
2. 累计请求次数
3. 最后请求时间
4. 最近基站信息
5. 最近定位结果

### 9.2 排位维度

建议记录：

1. 所属设备
2. 排位编号
3. 当前用户
4. 当前状态
5. 当前内容版本

### 9.3 内容维度

建议记录：

1. 图片文件
2. 音频文件
3. 上传时间
4. 上传用户
5. 内容版本
6. 是否已下发

## 10. 非功能要求

1. 设备请求接口应尽量稳定且响应快。
2. 文件上传应支持断点明确、失败可重试。
3. 内容文件应具备命名规范，避免冲突。
4. 管理端和用户端权限应隔离。
5. 系统应保留关键操作日志，便于追踪问题。

## 11. MVP 建议范围

第一阶段建议先实现以下最小可用能力：

1. 设备列表与设备详情
2. 排位与设备的绑定关系
3. 排位分配给最终用户
4. 最终用户上传图片和音频
5. 设备通过接口获取最新配置
6. 记录设备请求次数和最后请求时间

## 12. 后续可扩展方向

1. 设备分组管理
2. 内容审核机制
3. 内容版本回滚
4. 用户自助登录与通知机制
5. 数据统计报表
6. 设备异常告警
7. 位置信息地图展示

## 13. 总结

设备管理功能的核心，是围绕“祝福牌设备 - 排位 - 最终用户 - 内容资源”四个对象建立稳定的管理与同步链路。系统既要让管理员能高效管理设备和排位，也要让最终用户能简单地维护自己的内容，同时保证设备可以稳定获取并循环播放最新资源。
